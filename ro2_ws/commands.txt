To display the model generate by urdf file (not xacro) :
ros2 launch urdf_tutorial display.launch.py model:=<absolute_path/file_name.urdf>

To create a PDF containing the frames of the robot:
ros2 run tf2_tools view_frames

To display the model generate by xacro file and urdf.xacro (execute in ros2_ws):
1) Converts the xacro into a urdf file, giving the path to the urdf.xacro file
2) Launch rviz using the urdf file generated before
ros2 run xacro xacro -o /tmp/limo.urdf <relative path/file_name.urdf.xacro>
ros2 launch urdf_tutorial display.launch.py model:=/tmp/limo.urdf

To build a package:
colcon build --packages-select package_name
source install/setup.bash

To run an executable:
ros2 run package_name node_name

To modify when creating a new node:

package.xml 
<description>Examples of minimal publisher/subscriber using rclpy</description>
<maintainer email="you@email.com">Your Name</maintainer>
<license>Apache License 2.0</license>
<exec_depend>package</exec_depend> (example: <exec_depend>std_msgs</exec_depend>)

setup.py -> match previous fields: maintainer, maintainer_email, description and license
in console_scripts add:

entry_points={
        'console_scripts': [
                'node_name = package_name.node_name:main',
        ],
},
